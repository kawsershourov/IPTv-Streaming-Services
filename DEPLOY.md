# Deploying SunPlex to cPanel

The `ready-for-deploy/` folder is a **production-ready copy** (debug off, HTTPS `.htaccess`, dev files
removed). Build/refresh it with `bash build-deploy.sh`, then follow these steps.

> Domain root + HTTPS assumed (e.g. `https://yourdomain.com/`). The `.htaccess` forces HTTPS,
> so make sure SSL is installed first (cPanel → **SSL/TLS Status** → AutoSSL / Let's Encrypt).

## 1. Create the database (cPanel → MySQL® Databases)
1. Create a database, e.g. `cpaneluser_sunplex`.
2. Create a DB user with a strong password.
3. **Add the user to the database** with **All Privileges**.

## 2. Import the schema + data (cPanel → phpMyAdmin)
Select the database, then **Import**:
- `sql/schema.sql` (creates the tables), then
- `sql/seed.sql` (admin user, plans, demo categories/channels).

**Migrating your existing data instead?** On your local machine export the `sunplex` database
(phpMyAdmin → Export → Quick), and import that file here instead of `seed.sql`.

## 3. Configure
Edit `ready-for-deploy/app/config.php` and set your real `db.name`, `db.user`, `db.pass`.
(`base_url` is already empty for the domain root, `debug` is `false`, and a random `app_key`
was generated by the build.)

## 4. Upload
- Zip the **contents** of `ready-for-deploy/` (not the folder itself).
- cPanel → **File Manager** → `public_html` → **Upload** the zip → **Extract**.
  (Or upload via FTP.) The files must sit at the **public_html root**.

## 5. Permissions
- `uploads/` and `uploads/logos/` must be **writable** (755 or 775) so logo/import uploads work.

## 6. First run
1. Visit `https://yourdomain.com/` — the home player should load.
2. Log into `https://yourdomain.com/admin/` with `admin@sunplex.live` / `admin123`
   and **immediately change the password** (Admin → Users, or via the account).
3. Replace the placeholder stream URLs with your **licensed** streams (Admin → Channels,
   or bulk Import CSV/Excel).
4. Upload your logo + favicon (Admin → Settings) and paste your Google Analytics code
   (Admin → Settings → Tracking & custom code).

## Security checklist (already wired, verify after deploy)
- [x] HTTPS forced + HSTS (`.htaccess`)
- [x] Security headers, directory listing off, sensitive files blocked
- [x] Brute-force login lockout (6 failures / 15 min per IP or email)
- [x] HttpOnly + Secure + SameSite session cookies, session-fixation guard
- [x] CSRF on every form, PDO prepared statements (no SQL injection)
- [x] Gzip + 1-year asset caching (assets are `?v=` cache-busted)
- [ ] **Change the admin password**
- [ ] Keep PHP **8.1+** (set in cPanel → Select PHP Version; enable `pdo_mysql`, `zip`, `gd`)

## Notes
- Requires the Apache modules `mod_rewrite`, `mod_headers`, `mod_deflate`, `mod_expires`
  (standard on cPanel).
- The full official FWD UVP player ships under `/player`; to update it later, replace that
  folder with a newer official release.
