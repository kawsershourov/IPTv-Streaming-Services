# Deploying SunPlex to cPanel

The `ready-for-deploy/` folder is a **production-ready copy** (debug off, HTTPS `.htaccess`, dev files
removed). Build/refresh it with `bash build-deploy.sh`, then follow these steps.

> Domain root + HTTPS assumed (e.g. `https://yourdomain.com/`). The `.htaccess` forces HTTPS,
> so make sure SSL is installed first (cPanel → **SSL/TLS Status** → AutoSSL / Let's Encrypt).

## 1. Create the database (cPanel → MySQL® Databases)
1. Create a database, e.g. `cpaneluser_sunplex`.
2. Create a DB user with a strong password.
3. **Add the user to the database** with **All Privileges**.

## 2. Import the database (cPanel → phpMyAdmin)
- **Your data:** on your PC export the `sunplex` database (phpMyAdmin → Export → Quick → SQL),
  then **Import** that `.sql` file into the cPanel database.
- **Or a fresh start:** import `sql/schema.sql` then `sql/seed.sql`.

> If you uploaded logos/media locally, the stored URLs include `/SunPlex.live/...`. On the domain
> root, find-replace `/SunPlex.live/uploads` → `/uploads` in your export before importing (or in
> phpMyAdmin after).

## 3. Configure
Edit `ready-for-deploy/app/config.php` and set your real `db.name`, `db.user`, `db.pass`.
(`base_url` is already empty for the domain root, `debug` is `false`, and a random `app_key`
was generated by the build.)

## 4. Upload
- Zip the **contents** of `ready-for-deploy/` (not the folder itself).
- cPanel → **File Manager** → `public_html` → **Upload** the zip → **Extract**.
  The files must sit at the **public_html root**.

## 5. Permissions
- `uploads/`, `uploads/logos/`, `uploads/media/` must be **writable** (755 or 775).

## 6. First run
1. Visit `https://yourdomain.com/` — the home player should load.
2. Log into `/admin/` with `admin@sunplex.live` / `admin123` and **change the password**.
3. Replace placeholder stream URLs with your **licensed** streams (Admin → Channels / Import).
4. Upload your logo + favicon (Admin → Settings), paste Google Analytics (Settings → Tracking).

## Security checklist (already wired)
- [x] HTTPS forced + HSTS, security headers, no directory listing, sensitive files blocked
- [x] Uploads can't execute PHP (`uploads/.htaccess`)
- [x] Brute-force login lockout (6 fails / 15 min)
- [x] HttpOnly + Secure + SameSite cookies, session-fixation guard
- [x] CSRF on every form, PDO prepared statements (no SQL injection)
- [x] Stream URLs encoded out of the page source (Admin → Player → "Hide stream URLs")
- [x] Gzip + 1-year asset caching
- [ ] **Change the admin password**; keep PHP **8.1+** (`pdo_mysql`, `zip`, `gd`)

## Notes
Requires `mod_rewrite`, `mod_headers`, `mod_deflate`, `mod_expires` (standard on cPanel).
The actual stream request is still visible in the browser network tab — fully hiding it would need a
server proxy (heavy bandwidth) or DRM. Source encoding stops casual view-source scraping.
