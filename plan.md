# SunPlex.live — Build Plan

Standalone PHP + MySQL rebuild of the WordPress IPTV site `sunplex.live`, using the
FWD Ultimate Video Player **JS engine** for live HLS playback and a custom backend for
channels, categories, users, and subscription access.

## Decisions
- **Stack:** PHP 8 + MySQL (XAMPP / Apache).
- **Scope:** Core + Subscriptions — auth, categories, live channels, UVP player, admin CRUD,
  subscription plans with premium/expiry access control. **No payment gateway** yet (plans
  assigned manually in admin).
- **Version control:** local git, one commit per phase.
- **Design:** replicate the sunplex.live dark IPTV grid (category rows, channel logo tiles).

## Architecture
```
index.php category.php watch.php login.php register.php logout.php account.php   front-end
/app   config db.php auth.php access.php helpers.php /models /includes           backend (guarded)
/admin index dashboard categories channels users plans settings                  admin panel
/player js/FWDUVP.js css/fwduvp.css css/fwd_ui.css content/<skin>                 UVP JS engine
/assets css/site.css js/site.js img                                              theme
/sql   schema.sql seed.sql                                                       database
/uploads/logos                                                                   admin uploads
```

## Database
`users`, `plans`, `user_subscriptions`, `categories`, `channels`, `settings`, `password_resets`.
An active, non-expired `user_subscriptions` row ⇒ premium access. `channels.stream_type`
defaults to `hls` (`.m3u8`).

## Player integration
`watch.php` embeds the UVP engine directly:
```js
new FWDUVPlayer({
  instanceName: "player1", parentId: "myDiv",
  mainFolderPath: "/SunPlex.live/player/content", skinPath: "minimal_skin_dark",
  useHLS: true,
  playlists: [{ videoSource: [{ source: "<channel.stream_url>", label: "auto" }], title: "<name>" }]
});
```
Playlist/videoSource arrays are generated by PHP from the DB. Exact v11.0 prop names are verified
against `js/FWDUVP-unminified.js` during implementation.

## Build phases (one commit each)
1. Repo init — git, .gitignore, README, plan.md, session.md, config.sample.php.
2. Database — schema.sql + seed.sql.
3. App core — config, db (PDO), auth, access, helpers, models, front-end includes.
4. Player assets — extract UVP engine into /player.
5. Auth pages — register/login/logout/forgot-password/account.
6. Home + category — sunplex.live-style grid + site.css.
7. Watch page — UVP embed + can_watch() gating + channel rail.
8. Subscriptions — plans listing, account page, premium gating end-to-end.
9. Admin panel — login, dashboard, categories, channels, users, plans, settings.
10. Seed real data + polish + docs + session.md.

## Verification
Start XAMPP → import SQL → set config.php → browse `http://localhost/SunPlex.live/`.
Register/login, browse category rows, play an HLS channel in the UVP player, confirm premium
gating blocks free users, and exercise admin CRUD. `git log --oneline` shows per-phase history.

## Out of scope (later)
Payment gateway (Stripe/bKash), production hardening (tokenized streams, HTTPS, SMTP),
WordPress data migration.
